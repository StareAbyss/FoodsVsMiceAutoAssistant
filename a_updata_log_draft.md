# 重大更新
* 全新的智能战斗V3
  * 完全颠覆性重构旧版战斗方案, 和无脑的遍历SayGoodBey~
  * 战斗方案详解
    * 以[优先级队列] X [状态识别]为基础进行
    * 每张卡片具有 是否在冷却中/是否可用/是否被禁止使用三条属性
      * 是否在冷却中/是否可用, 由一秒扫描一次卡片的图像状态以实现 (具体实现为判定每张卡片位置范围后, 扫描其最顶和最低一排像素颜色)
      * 是否被静止使用, 则是当卡片使用后, 立即扫描该卡片, 如果扔处于可使用状态, 进入7s的禁止使用状态, 防止冷却时间较短的优先级较高的卡片放满后堵塞队列
    * 每张卡片将以设定的优先级, 即战斗方案在战斗方案编辑器中从上到下的顺序进行放卡
      * 以低频率(1Hz)检测
        * 图像识别所有卡片的状态并更新, 并全部重新装填回到队列中, 在队列中的顺序为优先级
        * 检测是否是战斗完成/需要点击开始关卡后半部分
      * 以高频率(50Hz)使用
        * 从队列中尝试使用队顶卡片, 如果该卡片处于可使用状态且未被禁止使用, 点击选中该卡片, 并进行放置的点击操作, 否则立刻从队列中移除
        * 使用卡片后, 将立刻对该卡片识图, 并更新其状态参数
  * 相比传统方案的优点
    * 面向对象的抽象, 具有高度可读性和拓展性
    * 能以优先级作为放卡的第一准则, 以防止低产能情况下防止低耗火卡被优先使用, 即放了一堆瓜皮布丁, 就是不放海星 orz
    * 能以动态的方式处理冷却时间(CD), 不再具有一轮放卡的概念, 也就是不会定死每一张卡只有7s的倍速的时间才会被放卡一次
      * 极大优化了极短CD的卡片, 即承载卡的放置速度
      * 极大优化了不规则长CD的卡片, 在不同练度情况下的适配度, 能尽可能完美利用每一秒CD
  * 无需在战斗方案添加, 仅需在卡组放置对应卡片, 即可全自动识别并以高度智能的手法放置以下卡片:
    * 极寒冰沙(冷却辅助)
      * 在所有卡片陷入冷却 或 被禁止使用状态时使用
      * 两个角色之间会自动岔开使用防止重复
      * * 该功能仅火苗能量 >= 1000时启动
      * 仅二转的极寒冰沙, 不支持未完成二转的冰激凌卡片
    * 幻幻鸡(复制类)
      * 在战斗方案中为所有卡片添加了幻幻鸡参数(坤参数), 该参数为0即不复制该卡片
      * 当有任意卡片坤参数 >= 1时, 启动该功能, 将自动复制目前所有卡片中 未被禁止使用(即未铺满), 且坤参数最高的卡片
      * 该功能仅火苗能量 >= 1000时启动
  * 补充说明
    * 承载卡和任务卡将以最高优先级被插入战斗方案中
    * 对于循环但不遍历的放卡(一般是棉花糖类承载卡), 现在每次放卡都会尝试点击两次, 以使其能稳定不漏的放下
* 全新界面!
  * 以多段标签页排布元素, 完全重绘了UI布局, 美观度↑↑↑
  * 使用SmileySans(得意黑)开源字体作为全新的字体!
    * 更窄的字宽适应, 提高文本内容密度.

# 修复
* 软件崩溃问题
  * 增强了游戏刷新后检测是否成功进入游戏的检测力度, 以防止误测. 起因是欢乐互娱更换了游戏登录页背景图后, 其中一部分与此前用于识别是否成功进入游戏标志图像高度相似, 导致即使没能成功进入游戏, 也会误认为进入了并对不存在的flash进行识图比较导致卡死报错, 通过更严格的多图像识别同时存在, 应该可以解决问题.
* 功能故障
  * 自动战斗
    * 修复了2P单人作战时不铲卡的问题
    * 此修正了铲卡铲漏的问题.
    * 修复了魔法软糖二转图标使用未转职图标, 导致战斗中会重复识别到两次魔法软糖不转, 导致承载卡漏放(被分配给了不存在的魔法软糖二转)
    * 将战斗中人物的放置步骤移动到扫描卡组初始化战斗方案前, 以解决部分机器卡顿的虚拟机偶尔无法正确放置人物的问题.
  * 全局点击队列线程 - 修正了点击中止按钮时该线程未中止的问题, 并重新整理了点击按钮动作绑定的槽函数, 以增加可读性
  * 全自动美食大赛
    * 修复了设置不使用钥匙的关卡未按预期直接结束的bug
    * 修复了只循环一轮且任务未能全部完成就立刻结束的bug
  * 老版本在 [先锋鼠军字样] 位置查找 [精英鼠军字样] 导致 [战斗中点击继续战斗按钮的函数] 一直未能正确执行的bug.
    * 老版本能相对正常点击继续作战, 是由于一般战斗方案是从左到右从上到下遍历放卡的, 而其中4-2 4-3位置正好可以点到继续作战按钮, 4-4 4-5位置正好可以点到[领取奖励按钮], 因此依靠该bug形成了大多数情况下能继续作战, 极少部分情况下会点到领取奖励的情况.
      * 为彻底修正该bug, 做了如下修改:
        * 将第四行的点击坐标的y轴向上移动约20个像素
          * 在不影响放卡的同时抹除了放卡点到按钮的可能性.
          * 修正了在放卡流程期间弹出 [是否继续战斗UI] 时可能出现放卡错乱的问题.
        * 修正了查找位置作物的问题, 并改为在查找到一次[精英鼠军字样后], 将循环查找点击直到不再显示[是否继续作战UI]退出使用钥匙模式.
        * 取消了可能点到[继续作战or领取奖励字样]的位置, 在点击该位置时额外检查的设定, 因为坐标已偏移.
  * 自动使用消耗品 - 修复部分消耗品无法识别的BUG
