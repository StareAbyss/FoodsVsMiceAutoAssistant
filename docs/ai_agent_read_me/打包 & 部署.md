# 打包与部署

本文说明发布包的打包、安装、调试和卸载流程。源码开发运行方式见 `运行环境和调试环境.md`。

## 打包

发行打包应从干净的 tag / PR merge commit 工作区执行。推荐在项目根目录上一级使用：

```text
_biuld/
```

作为临时构建工作区。打包结束后，`_biuld/` 内的临时 worktree 或源码拷贝可以删除。

发行产物输出到项目根目录上一级：

```text
_dist/
```

`_dist/` 是产物归档区，只能新增，打包脚本不会删除旧产物。每次打包会新建类似下面的目录：

```text
FAA-v3.1.1 2026-06-07 15-39-33
```

Windows 文件夹名不能包含英文冒号，因此时间使用 `HH-mm-ss`。

在要打包的干净源码根目录运行：

```powershell
python 一键生成分发资源.py
```

如当前 shell 没有可用 Python，先按 `运行环境和调试环境.md` 配好 `.venv` 或使用：

```powershell
uv run python 一键生成分发资源.py
```

如果从开发工作区运行脚本，但希望打包 `_biuld/` 中的干净源码，可以设置：

```powershell
$env:FAA_PACKAGE_SOURCE_ROOT = "F:\...\_biuld\FAA-v3.1.1"
uv run python 一键生成分发资源.py
```

脚本会检查 `FAA_PACKAGE_SOURCE_ROOT` 对应源码是否处于带 tag 的 GitHub PR merge commit，检查不通过会中断。

## 普通用户安装入口

发布包中普通用户运行：

```text
AppInstallRun.bat
```

源码仓库中的脚本位置：

```text
plugins\uv\AppInstallRun.bat
```

脚本行为：

- 自动定位 `pyproject.toml` 所在目录。
- 使用项目内受控的 `bin\uv.exe`，不优先使用系统 PATH 中的 uv。
- 如果本地 uv 不存在，从内置 zip 安装到脚本目录下的 `bin`。
- 安装 uv 前校验内置 zip 的 SHA256。
- 设置 `UV_INSTALL_DIR`、`UV_NO_MODIFY_PATH=1`、`UV_DISABLE_UPDATE=1`，不修改用户 PATH。
- 执行 `uv python install 3.12`。
- 执行 `uv sync --locked`，按 `uv.lock` 恢复依赖。
- 启动 `python -m function.faa_main`。

脚本内部已拆分为：

```text
FindAppDir
PrepareUv
InstallPython
SyncLocked
RunApp
Abort
```

这样安装失败时可以快速定位卡在哪一步。

## 内置 uv 包

```text
文件: plugins\uv\uv\uv-x86_64-pc-windows-msvc.zip
版本: uv 0.6.12 (e4e03833f 2025-04-02)
来源: https://github.com/astral-sh/uv/releases/tag/0.6.12
SHA256: 30FDF26C209F0CB7C97D3B08A26AB4E78CE5AE0E031B88798CBACCC0F24F452B
```

如果 zip 被替换或损坏，`AppInstallRun.bat` 会因 SHA256 不匹配而停止。

## 正式版启动

安装完成后运行：

```text
FAA.exe
```

## 调试版启动

安装完成后运行：

```text
debug_run.bat
```

源码仓库中的脚本位置：

```text
plugins\uv\debug_run.bat
```

`debug_run.bat` 使用项目内 `bin\uv.exe`，不会优先使用系统 PATH 中的 uv。首次运行前应先执行 `AppInstallRun.bat`，确保本地 uv 和依赖环境已经准备好。

## 卸载入口

源码仓库中的脚本位置：

```text
plugins\uv\UVFAAUninstaller.bat
```

该脚本只删除 FAA 管理的本地目录：

```text
plugins\uv\bin
.venv
```

不会删除用户全局 uv、uv cache、uv-managed Python，也不会清空脚本目录。

## 依赖和镜像策略

`pyproject.toml` 当前使用清华镜像作为默认 index：

```toml
[[tool.uv.index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true
```

`uv.lock` 中的版本和 hash 是依赖完整性的依据。镜像源可以按网络情况调整，但发布安装流程不应重新解析依赖或随意改写 `uv.lock`。

发布或用户安装入口必须使用：

```powershell
uv sync --locked
```

开发者需要更新依赖时，在开发环境中执行 `uv add`、`uv remove` 或 `uv sync`，确认结果后提交 `pyproject.toml` 和 `uv.lock`。

## 系统要求

由于 2.2 版本引入 `onnxruntime-directml`，不再需要 OpenCV GPU 推理框架。电脑环境需注意 Windows 10 版本大于等于 2004；Windows 11 默认支持。
