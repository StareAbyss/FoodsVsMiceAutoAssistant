# 打包与部署

本文说明发布包的打包、安装、调试和卸载流程。源码开发运行方式见 `运行环境和调试环境.md`。

## 打包

发行打包应从干净的 tag / PR merge commit 工作区执行。推荐在项目根目录上一级使用：

```text
_biuld/
```

作为临时构建工作区。打包结束后，`_biuld/` 内的临时 worktree 或源码拷贝可以删除。

发行产物输出到项目根目录上一级：

```text
_dist/
```

`_dist/` 是产物归档区，只能新增，打包脚本不会删除旧产物。每次打包会新建类似下面的目录：

```text
FAA-v3.1.1 2026-06-07 15-39-33
```

Windows 文件夹名不能包含英文冒号，因此时间使用 `HH-mm-ss`。

在要打包的干净源码根目录运行：

```powershell
python 一键生成分发资源.py
```

如当前 shell 没有可用 Python，先按 `运行环境和调试环境.md` 配好 `.venv` 或使用：

```powershell
uv run python 一键生成分发资源.py
```

如果从开发工作区运行脚本，但希望打包 `_biuld/` 中的干净源码，可以设置：

```powershell
$env:FAA_PACKAGE_SOURCE_ROOT = "F:\...\_biuld\FAA-v3.1.1"
uv run python 一键生成分发资源.py
```

脚本会检查 `FAA_PACKAGE_SOURCE_ROOT` 对应源码是否处于带 tag 的 GitHub PR merge commit，检查不通过会中断。

## 普通用户启动入口

发布包中普通用户运行：

```text
FAA.exe
```

`FAA.exe` 是极薄的启动器，只负责调用 `plugins\launcher_scripts\AppInstallRun.bat`。不要让 `FAA.exe` 直接寻找系统 PATH 中的 uv，否则会破坏开袋即食目标。

启动器源码和构建脚本位于：

```text
tool\packaging\faa_launcher\FAA.cs
tool\packaging\build_faa_launcher.ps1
```

需要重新生成启动器时运行：

```powershell
powershell -ExecutionPolicy Bypass -File tool\packaging\build_faa_launcher.ps1
```

## 安装和运行脚本

`FAA.exe` 会调用：

```text
plugins\launcher_scripts\AppInstallRun.bat
```

`AppInstallRun.bat` 只保留 ASCII 启动壳，避免 `cmd.exe` 解析 UTF-8 中文 batch 时出现乱码或把中文行误当作命令。它是 `FAA.exe` 的内部跳板，不作为根目录用户入口暴露。真正的安装、提示和启动逻辑在：

```text
plugins\launcher_scripts\AppInstallRun.ps1
```

源码仓库中的脚本位置：

```text
plugins\launcher_scripts\AppInstallRun.bat
plugins\launcher_scripts\AppInstallRun.ps1
```

脚本行为：

- 自动定位 `pyproject.toml` 所在目录。
- 使用项目内受控的 `bin\uv.exe`，不优先使用系统 PATH 中的 uv。
- 如果本地 uv 不存在，从内置 zip 安装到脚本目录下的 `bin`。
- 安装 uv 前校验内置 zip 的 SHA256。
- 设置 `UV_INSTALL_DIR`、`UV_NO_MODIFY_PATH=1`、`UV_DISABLE_UPDATE=1`，不修改用户 PATH。
- 执行 `uv python install 3.12`。
- 执行 `uv sync --locked --no-dev`，按 `uv.lock` 恢复普通用户运行依赖，不安装开发依赖。
- 执行 `.venv\Scripts\pythonw.exe -m function.faa_main` 独立启动主程序，启动后安装黑框停留数秒并自动关闭。

脚本内部已拆分为：

```text
FindAppDir
PrepareUv
InstallPython
SyncLocked
RunApp
Abort
```

这样安装失败时可以快速定位卡在哪一步。

## 内置 uv 包

```text
文件: plugins\launcher_scripts\uv\uv-x86_64-pc-windows-msvc.zip
版本: uv 0.6.12 (e4e03833f 2025-04-02)
来源: https://github.com/astral-sh/uv/releases/tag/0.6.12
SHA256: 30FDF26C209F0CB7C97D3B08A26AB4E78CE5AE0E031B88798CBACCC0F24F452B
```

如果 zip 被替换或损坏，`plugins\launcher_scripts\AppInstallRun.bat` 会因 SHA256 不匹配而停止。

## 正式版入口约束

正式版入口仍然是：

```text
FAA.exe
```

`FAA.exe`、`FAA-调试模式启动.bat`、`FAA-卸除运行环境.bat`、`FAA-恢复到备份.bat`、`plugins\launcher_scripts\AppInstallRun.bat`、`plugins\launcher_scripts\AppInstallRun.ps1`、`plugins\launcher_scripts\UVFAAUninstaller.ps1`、`pyproject.toml` 和 `uv.lock` 都属于发布包关键文件，热更新 staging 校验必须保证它们存在。

## 带运行框启动

需要排查启动问题时运行：

```text
FAA-调试模式启动.bat
```

源码仓库中的脚本位置：

```text
plugins\root_entries\FAA-调试模式启动.bat
```

该入口和 `FAA.exe` 使用同一套 `AppInstallRun.ps1` 流程，会照常准备 uv、Python 和依赖。区别只在最后启动主程序时使用 `.venv\Scripts\python.exe` 在当前黑框中前台运行 `python -m function.faa_main`，用于查看启动日志、异常 traceback 和退出码。

## 卸载入口

源码仓库中的脚本位置：

```text
plugins\root_entries\FAA-卸除运行环境.bat
plugins\launcher_scripts\UVFAAUninstaller.ps1
```

该脚本只删除 FAA 管理的本地目录：

```text
plugins\launcher_scripts\bin
.venv
```

不会删除用户全局 uv、uv cache、uv-managed Python，也不会清空脚本目录。

## 依赖和镜像策略

`pyproject.toml` 当前使用清华镜像作为默认 index：

```toml
[[tool.uv.index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true
```

`uv.lock` 中的版本和 hash 是依赖完整性的依据。镜像源可以按网络情况调整，但发布安装流程不应重新解析依赖或随意改写 `uv.lock`。

发布或用户安装入口必须使用：

```powershell
uv sync --locked --no-dev
```

开发者需要更新依赖时，在开发环境中执行 `uv add`、`uv remove` 或 `uv sync`，确认结果后提交 `pyproject.toml` 和 `uv.lock`。

## 系统要求

由于 2.2 版本引入 `onnxruntime-directml`，不再需要 OpenCV GPU 推理框架。电脑环境需注意 Windows 10 版本大于等于 2004；Windows 11 默认支持。
