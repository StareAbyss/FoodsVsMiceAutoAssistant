# 全局路径和资源管理

本文记录本仓库路径拼接、资源定位和资源加载的统一规则。

## 路径来源

* 项目根路径和常用目录统一从 `function.globals.get_paths.PATHS` 读取。
* 新代码不要自行用 `__file__`、当前工作目录或硬编码绝对路径推导项目根目录。
* 需要新增常用全局目录时，优先在 `build_paths()` 中补充 `PATHS` 项，再在业务代码中引用。

## 路径拼接

* 新代码统一使用 `os.path.join()` 拼接路径。
* 不要使用字符串加法拼接路径，例如：

```python
PATHS["root"] + "\\config\\settings.json"
PATHS["config"] + "//stage_info.json"
PATHS["logs"] + "\\match_failed"
```

* 推荐写法：

```python
os.path.join(PATHS["config"], "settings.json")
os.path.join(PATHS["config"], "stage_info.json")
os.path.join(PATHS["logs"], "match_failed")
```

* 当前仓库的 `PATHS` 值保持为 `str`。暂不全局改成 `pathlib.Path`，避免 PyQt、OpenCV、PIL、打包脚本等调用点出现兼容性差异。
* 如果局部确实需要使用 `pathlib.Path`，传入 PyQt、OpenCV、PIL、`json/open`、`sqlite` 等外部库前优先显式转换为 `str(path)`。

## 资源加载

* 资源文件优先通过已有全局资源表读取，例如 `RESOURCE_P`、`PATHS["image"]`、`PATHS["ui"]`、`PATHS["logo"]`。
* 不要在业务代码中重复手写 `resource\\image\\...`、`resource\\ui\\...` 等长路径。若资源目录有稳定复用价值，应补充到 `PATHS` 或资源索引中。
* 加载配置、数据库、图片、UI 文件时，应先用 `PATHS` 定位目录，再用 `os.path.join()` 追加文件名。

示例：

```python
ui_path = os.path.join(PATHS["ui"], "FAA_3.0.ui")
logo_path = os.path.join(PATHS["logo"], "圆角-FetDeathWing-256x-AllSize.ico")
db_path = os.path.join(PATHS["db"], "tasks.db")
```

## 修改旧代码

* 修旧代码时，优先把同一文件内的 `PATHS[...] + "\\xxx"`、`PATHS[...] + "//xxx"` 改为 `os.path.join()`。
* 路径统一化应保持行为不变，不顺手移动文件、不重命名资源、不调整资源索引结构。
* 大范围清理时分批提交，避免路径改动和业务逻辑改动混在一起。
