# 运行环境和调试环境

本文说明开发者和 AI 智能体在源码仓库中如何运行、调试和维护依赖。发布包安装流程见 `打包 & 部署.md`；JetBrains/Codex 的 PATH 排障见 `AI运行环境配置.md`。

## 当前项目环境

项目已切到：

```text
pyproject.toml + uv.lock + .venv
```

当前 `.venv`：

```text
路径: F:\My Project\Python\FoodsVsMousesAutoAssistant\.venv
Python: 3.12.9
来源: uv-managed CPython
```

项目要求：

```toml
requires-python = ">=3.12"
```

因此 `.venv` 使用 Python 3.12.9 是正常的；不要再使用旧的 `F:\My RTE\Python\Venv\FoodsVsMousesAutoAssistant` 作为默认开发环境。

项目模型已使用 `pyproject.toml`。如果 PyCharm 提示解释器异常，优先确认解释器是否仍指向上面的 `.venv`。

## 常用命令

同步依赖：

```powershell
uv sync --locked
```

运行主程序：

```powershell
uv run python -m function.faa_main
```

快速编译检查：

```powershell
uv run python -m compileall -q function plugins
```

如果当前 shell 的 PATH 异常，可用绝对路径：

```powershell
& 'D:\Dev\Tools\uv\uv.exe' sync --locked
& 'D:\Dev\Tools\uv\uv.exe' run python -m function.faa_main
& 'D:\Dev\Tools\uv\uv.exe' run python -m compileall -q function plugins
```

## 依赖维护

依赖入口是 `pyproject.toml`，锁定结果是 `uv.lock`。

添加普通依赖：

```powershell
uv add requests
```

添加开发依赖：

```powershell
uv add --dev pyinstaller
```

删除依赖：

```powershell
uv remove requests
```

更新依赖后需要提交：

```text
pyproject.toml
uv.lock
```

发布或用户安装入口必须使用 `uv sync --locked`，避免在用户机器上重新解析依赖。

## 注意事项

- 当前依赖以 `pyproject.toml` 为准，不要新增 requirements 文件作为第二套依赖来源。
- 不要看到 `python`、`pip`、`uv` 找不到就判断未安装，先检查 PATH 或使用绝对路径。
- `pytest` 当前不是项目 dev 依赖；如果需要引入测试框架，应通过 `uv add --dev pytest` 添加。
- `UV_LINK_MODE=copy` 用于避免项目在 `F:` 盘、uv cache 在 `D:` 盘时跨盘 hardlink 警告。
