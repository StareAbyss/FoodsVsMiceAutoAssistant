# 测试目录约定

`test/` 只保存能够帮助后续开发者复现、验证和审阅项目行为的内容。

## 应提交到 Git

- 确定性单元测试和回归测试。
- 可重复执行的集成测试、性能基准和诊断脚本。
- 测试所需且经过筛选的最小输入夹具。
- 解释测试目标、运行条件和结果判定方法的文档。

## 不应提交到 Git

- 测试运行生成的截图、CSV、日志和性能报告。
- 只适用于当前电脑、账号或游戏窗口的临时研究结果。
- 外部 DLL、可执行文件和可以重新下载的第三方二进制。
- 没有复现价值的语言实验、一次性排查脚本和中间产物。

这些本地内容保存在对应测试旁边的 `output/` 目录，例如：

```text
test/wb_stage_recognition/output/
test/match_png/output/
```

上述目录中的图片、CSV、日志和数组结果由 `.gitignore` 按文件类型排除。截图和分析脚本应通过
`test.output_paths.get_test_output_dir()` 获取输出目录。

## 包结构

- `test/` 及所有包含 Python 模块的子目录都应保留 `__init__.py`。
- 新测试应按功能建立子目录，不继续向 `test/` 根目录平铺文件。
- 自动测试文件使用 `test_*.py`，人工诊断和性能基准应在文件名或文档中明确标识。
- 测试生成文件统一放在对应测试子目录的 `output/` 中。
- 测试不得被 `function/` 中的生产代码导入。
