# 战利品识别速度测试

本目录用于存放战利品识别性能验证脚本和阶段性结论。这里的内容是测试与方案验证，不属于 `tool/` 中可复用的开发工具。

## 测试脚本

```powershell
uv run python test\loot_recognition_speed\benchmark_loot_recognition_speed.py --limit 50 --repeat 1
```

默认读取：

- 战利品整板截图：`logs/loots_image/*.png`
- 战利品模板资源：`resource/image/item/战利品`
- DAG 排序数据：`config/item_ranking_dag_graph.json`

脚本中的老方案基准只提取识别逻辑，不保存截图，也不写入 `logs/match_failed`。

## DAG 的作用

游戏的战利品掉落展示存在稳定顺序，因此识别时不能把每个格子当作完全无序的独立样本。

当前逻辑通过 `item_ranking_dag_graph.json` 记录历史掉落先后关系，并使用 DAG ranking 作为候选匹配顺序。整板识别时会先尝试上一个识别结果，再沿 ranking 继续向后查找，最后才回退到全量模板遍历。这个顺序优化是当前识别速度的核心来源。

## 测试方案

| 方案 | 说明 |
|---|---|
| 老识别逻辑，无保存 | 保留当前 `one_item_match`、DAG ranking、`last_name` 逻辑，但去掉截图保存和失败块写入副作用。 |
| 预处理 `matchTemplate` | 提前缓存候选模板裁剪图、最终 mask 和绑定叠加图，仍使用 OpenCV 模板匹配。 |
| 纯数组相等 | 按道具 alpha 与掩模 alpha 生成 0/1 特征掩模，对源图和参考图乘掩模后做数组完全相等。 |
| 数组相等优先，失败回退模板 | 先用数组相等快速短路，失败后回退到预处理 `matchTemplate`。 |

## 精确数组方案规则

绑定判断：

1. 裁剪左下角绑定角标区域。
2. 使用 `物品-掩模-绑定.png` 中白色且不透明的像素作为必须识别区域。
3. 与 `物品-绑定角标-战利品.png` 的对应像素做数组相等。

物品判断：

1. 道具资源自身 `alpha == 255` 的像素是候选识别区域。
2. 当前模式对应掩模中 `alpha != 0` 的像素全部排除，用于避开绑定角标和掉落数量数字占位。
3. 得到 0/1 三通道特征掩模。
4. 对待识别图块和参考图分别乘特征掩模，再比较 BGR 数组是否完全一致。

## 2026-06-26 测试结果

测试样本：`logs/loots_image` 前 50 张 245x490 战利品整板截图。

### 替换 `None-0` 模板前

| 方案 | 板/秒 | 物品/秒 | 平均每板 | 与老方案差异 |
|---|---:|---:|---:|---:|
| 老识别逻辑，无保存 | 3.98 | 144.99 | 251.18 ms | 基准 |
| 预处理 `matchTemplate` | 6.42 | 233.68 | 155.85 ms | 0 |
| 纯数组相等 | 5.02 | 250.83 | 199.34 ms | 31 |
| 数组相等优先，失败回退模板 | 13.95 | 507.97 | 71.69 ms | 0 |

### 替换 `None-0` 模板后

`None-0.png` 已替换为真实截图中最常见的空格变体。此前抽样 61 个 `None-0` 中，该变体出现 59 次，占 96.72%。第三种明显变体来自游戏子弹图标遮挡，不应作为模板。

| 方案 | 板/秒 | 物品/秒 | 平均每板 | 与老方案差异 |
|---|---:|---:|---:|---:|
| 老识别逻辑，无保存 | 3.88 | 141.31 | 257.72 ms | 基准 |
| 预处理 `matchTemplate` | 6.32 | 230.20 | 158.21 ms | 0 |
| 纯数组相等 | 22.90 | 834.09 | 43.66 ms | 0 |
| 数组相等优先，失败回退模板 | 15.50 | 564.68 | 64.49 ms | 0 |

## 结论

替换 `None-0` 模板后，纯数组相等在当前 50 张样本中已能与老方案完全一致。

生产逻辑仍采用“数组相等短路 + 预处理模板匹配兜底”。原因是实际游戏截图可能出现子弹图标遮挡、边缘阴影浮动或少量像素扰动，模板兜底可以保持老方案的容错能力。

其中纯数组相等的 1 处差异为 `礼盒` 漏识别，说明精确数组路径仍可能受遮挡、边缘扰动或模板像素差异影响。生产路径使用“精确数组优先 + 模板兜底”，与老方案保持完全一致。

### 空模板场景限制

生产识别已把空物品模板按场景隔离：

- 战利品结算只允许 `None-0`、`None-1` 参与空格识别和提前终止。
- 翻宝箱只允许 `None-2`、`None-3`、`None-4`、`None-5`、`None-6` 参与空格识别和提前终止。

这样可以避免战利品截图命中宝箱背景空模板，也避免宝箱截图命中战利品结算空模板。测试脚本同步使用对应集合，当前 50 张战利品回归仍保持 `mismatches=0`，宝箱样本的 5 种空格变体全部落在 `None-2..6`。
